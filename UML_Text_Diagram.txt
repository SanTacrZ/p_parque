================================================================================
                              RELACIONES ENTRE CLASES
================================================================================

1. PARQUE → ATRACCIÓN (1:N)
   - Un parque contiene múltiples atracciones
   - Relación de composición

2. PARQUE → TAQUILLA (1:N)
   - Un parque tiene múltiples taquillas
   - Relación de composición

3. PARQUE → REGISTRO (1:N)
   - Un parque mantiene múltiples registros
   - Relación de composición

4. PARQUE → MANILLA (1:N)
   - Un parque genera múltiples manillas
   - Relación de composición

5. ATRACCIÓN → REGISTRO (1:N)
   - Una atracción genera múltiples registros
   - Relación de asociación

6. MANILLA → REGISTRO (1:N)
   - Una manilla puede generar múltiples registros
   - Relación de asociación

7. TAQUILLA → MANILLA (1:N)
   - Una taquilla vende múltiples manillas
   - Relación de asociación

8. PROGRAM → PARQUE (1:1)
   - El programa principal crea una instancia del parque
   - Relación de dependencia

================================================================================
                              FLUJO DE OPERACIONES
================================================================================

1. INICIALIZACIÓN:
   Program → Parque → Generar 1000 manillas
   Parque → Crear 3 taquillas con 100 manillas cada una

2. VENTA DE MANILLA:
   Visitante → Taquilla → Validar dinero
   Taquilla → Manilla → Asignar puntos
   Taquilla → Visitante → Entregar manilla

3. USO DE ATRACCIÓN:
   Visitante → Atracción → Verificar saldo
   Atracción → Manilla → Descontar puntos
   Atracción → Registro → Crear registro
   Atracción → Parque → Agregar registro

4. CONSULTA DE SALDO:
   Visitante → Manilla → Obtener saldo

5. RECARGA DE PUNTOS:
   Visitante → Taquilla → Validar dinero
   Taquilla → Manilla → Agregar puntos
